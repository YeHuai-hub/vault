# Vault — Windows 桌面密码管理器 实施计划

## Context

用户需要做一个全新的可安装分发的 Windows 桌面账号管理应用。旧版（account-manager-app）是 Electron 原型，用户对它各方面都不满意。这次从零开始，用更现代、更安全的技术栈重做，同时保留旧项目不动、不参考。

最终交付物：Windows .msi 安装程序，用户可双击安装、像普通软件一样使用和卸载。

## Tech Stack

| 层 | 技术 | 理由 |
|---|---|---|
| 桌面框架 | Tauri 2.x | 安装包 ~5MB，复用系统 WebView，Rust 沙箱安全 |
| 前端 | React 18 + TypeScript | 用户已接触过，生态成熟 |
| 构建 | Vite | Tauri 官方推荐，HMR 快 |
| 数据库 | SQLite (rusqlite) | 嵌入式、无需安装服务、断电不丢 |
| 加密 | Rust aes-gcm + pbkdf2 crates | 密钥永不到达 JS 层 |
| 安装包 | Tauri NSIS bundle | 生成 .msi 安装程序 |
| UI | Tailwind CSS | 快速出暗色风格 UI |

## 安全模型

```
用户输入主密码
    ↓
PBKDF2 (600k iterations, SHA-256) → 256-bit 密钥
    ↓
密钥仅驻留在 Rust 内存中（State 管理）
    ↓
写入：前端传明文 → Rust 加密(AES-256-GCM) → 存 SQLite
读取：前端请求 → Rust 从 SQLite 读密文 → 解密 → 返回明文给前端
    ↓
锁定：Rust 侧丢弃密钥 → 内存中无解密能力
```

**关键原则**：加密密钥终生不出 Rust 层。前端看到的永远是明文（解锁后）或空（锁定后）。前端不知道密钥存在形式。

- 主密码验证：注册时存 `PBKDF2(master_password, salt)` 的 hash + salt，解锁时重新计算对比
- 敏感数据加密：每个 entry 的 username/password/notes/custom_fields → JSON → AES-256-GCM(random nonce) → ciphertext 存 SQLite
- 自动锁定：默认 5 分钟无操作，可在设置中调整。计时器在前端，超时后调用 Rust `lock_vault` 命令
- 剪贴板安全：复制密码后 30 秒自动清除剪贴板

## Database Schema (SQLite)

```sql
-- 明文存储：用于分类、搜索、过滤
CREATE TABLE categories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  icon TEXT,
  sort_order INTEGER DEFAULT 0
);

CREATE TABLE entries (
  id TEXT PRIMARY KEY,           -- UUID v4
  category_id INTEGER,
  title TEXT NOT NULL,           -- 明文，用于搜索
  website TEXT,                  -- 明文
  encrypted_data TEXT NOT NULL,  -- AES-256-GCM 加密的 JSON:
                                 -- {username, password, notes, custom_fields: [{name, value}]}
  is_favorite INTEGER DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (category_id) REFERENCES categories(id)
);

CREATE TABLE tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE
);

CREATE TABLE entry_tags (
  entry_id TEXT NOT NULL,
  tag_id INTEGER NOT NULL,
  PRIMARY KEY (entry_id, tag_id),
  FOREIGN KEY (entry_id) REFERENCES entries(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);

CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL           -- JSON value
);
```

## Vault File

SQLite 数据库文件存于 Tauri 的 app data 目录：`%APPDATA%\com.vault.app\vault.db`

不存全局系统目录里，和 app 绑定，卸载即清空。

主密码验证 hash 单独存在 alongside db：`%APPDATA%\com.vault.app\vault.auth`

### Commands (Tauri IPC)

**Vault 生命周期：**
- `create_vault(master_password: &str) -> Result<()>` — 生成 salt，PBKDF2 导出密钥，存验证 hash，初始化 db 表
- `unlock_vault(master_password: &str) -> Result<()>` — 验证密码，加载密钥到内存
- `lock_vault() -> Result<()>` — 丢弃密钥
- `change_master_password(old: &str, new: &str) -> Result<()>` — 验证旧密码，重新加密所有数据
- `vault_exists() -> Result<bool>` — 检查是否已有 vault
- `vault_status() -> Result<VaultStatus>` — locked / unlocked / not_created

**Entries CRUD：**
- `list_entries(category_id: Option<i64>, search: Option<&str>, tag_id: Option<i64>) -> Result<Vec<Entry>>`
- `get_entry(id: &str) -> Result<Entry>`
- `create_entry(entry: CreateEntry) -> Result<Entry>`
- `update_entry(id: &str, entry: UpdateEntry) -> Result<Entry>`
- `delete_entry(id: &str) -> Result<()>`
- `toggle_favorite(id: &str) -> Result<Entry>`

**分类 & 标签：**
- `list_categories() -> Result<Vec<Category>>`
- `create_category(name: &str, icon: Option<&str>) -> Result<Category>`
- `delete_category(id: i64) -> Result<()>`
- `list_tags() -> Result<Vec<Tag>>`
- `create_tag(name: &str) -> Result<Tag>`
- `delete_tag(id: i64) -> Result<()>`

**工具：**
- `generate_password(length: u8, uppercase: bool, digits: bool, symbols: bool) -> Result<String>`
- `copy_to_clipboard(text: &str, auto_clear_secs: u64) -> Result<()>`
- `export_vault(path: &str, format: &str) -> Result<()>` — 加密 JSON / 明文 CSV
- `import_vault(path: &str, format: &str) -> Result<()>`

### State Management

用 React Context，三层：

```
VaultContext    — vault_exists, is_locked, unlock(), lock()
EntryContext    — entries[], selected_entry, CRUD 操作
UIContext       — sidebar_open, search_query, active_category, active_tag
```

不需要 Redux/Zustand，这个规模 Context 足够了。

### Routes

不用 React Router。单页应用，三个视觉状态：

1. **创建 vault**（vault 不存在）
2. **解锁**（vault 存在但 locked）
3. **主界面**（unlocked）

用 `VaultGate` 组件根据 `VaultContext` 状态条件渲染。

### Tailwind CSS

暗色主题为默认，后续可加亮色切换。配色方案：
- 背景：slate-900/950
- 卡片：slate-800
- 主色：cyan-500/600
- 文字：slate-100/300/500
- 危险：red-400

## 实施顺序 (Milestones)

### M1: 环境搭建 + Tauri 能跑
1. 安装 Rust（`rustup`）
2. 安装 Tauri CLI
3. `npm create tauri-app` 初始化项目
4. 安装 Tailwind CSS
5. 确认 `cargo tauri dev` 能弹出窗口
6. **验证**：白窗口出现 × 无 crash

### M2: 安全核心
1. 实现 Rust crypto 层（PBKDF2 + AES-256-GCM）
2. 实现 vault 创建/解锁/锁定命令
3. 创建 SQLite 数据库和表结构（迁移）
4. AppState 管理：encryption_key 的加载和丢弃
5. **验证**：创建 vault → 锁定 → 用正确/错误密码解锁 × 各场景

### M3: Entry CRUD
1. Rust 层条目增删改查命令（含加解密）
2. 前端 VaultGate + CreateVaultScreen + UnlockScreen
3. 前端 MainLayout + EntryList + EntryCard（只读视图）
4. 前端 EntryForm（创建/编辑）
5. **验证**：创建条目 → 密码明文入库（已验证加密）→ 编辑 → 删除 × 完整流程

### M4: 分类、标签、搜索
1. Rust 层分类/标签命令
2. 前端 Sidebar + CategoryList + TagSelector
3. 前端 SearchBar（带 debounce 的搜索）
4. **验证**：按分类过滤、按标签筛选、搜索模糊匹配

### M5: 密码生成器 + 剪贴板 + 自动锁定
1. Rust 密码生成命令
2. 前端 PasswordGenerator 组件
3. 剪贴板复制 + 30s 自动清除
4. 前端空闲检测 → 自动调用 lock_vault
5. **验证**：生成密码 → 复制 → 30s 后粘贴为空 ✓；闲置 5 分钟 → 锁定 ✓

### M6: 导入导出 + 安装包
1. 导出加密 JSON / 明文 CSV
2. 导入加密 JSON / 明文 CSV
3. 配置 Tauri NSIS bundler → 生成 .msi
4. **验证**：导出 → 删库 → 导入 → 数据完全恢复 ✓；msi 安装到另一台电脑可运行

### M7: 润色
1. 暗色 UI 风格定型
2. 键盘快捷键（Ctrl+N 新建、Ctrl+F 搜索、Ctrl+L 锁定）
3. 主密码修改功能
4. 收尾 CLAUDE.md

## 前提条件

执行 M1 前需要安装：
- **Rust**: `https://rustup.rs` → 下载 `rustup-init.exe` 运行（默认配置即可）
- 安装完成后重启终端，验证 `rustc --version` 和 `cargo --version`

## Verification

每个 milestone 完成后手动验证：
1. `cargo tauri dev` 启动开发模式
2. 按 milestone 的验证清单逐项操作
3. 检查 Rust 侧无明显 unwrap panic（用 `cargo tauri build --debug` 检查编译警告）
4. 前端 Console 无红色报错
